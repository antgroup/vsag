# VSAG 1.0 Release Notes

`v1.0.0` was released on July 12, 2026.

- [Official GitHub Release](https://github.com/antgroup/vsag/releases/tag/v1.0.0)
- [Full v0.18.0...v1.0.0 changelog](https://github.com/antgroup/vsag/compare/v0.18.0...v1.0.0)
- Tag commit: `efdaf17a10e96cdb5222baf558d50dfacbdc672e`

## Overview

VSAG 1.0 is the project's first stable major release. The public `0.x` history spans 81 tags
across the `v0.11`-`v0.18` maintenance lines. The `v1.0.0` tag captures the final mainline state
as a broad vector indexing toolkit for dense, sparse, structured-filter, hierarchical, and
multi-vector retrieval.

The official `v1.0.0` release groups the work since `v0.18.0` into 375 entries: 48 new features,
134 improvements, 105 bug fixes, and 88 other changes. The `v0.11.0...v1.0.0` mainline comparison
contains 1,252 commits.

This editorial release note describes the final state of the `v1.0.0` tag. It intentionally omits
intermediate features that were added and later removed before the release.

## What VSAG 1.0 Delivers

### Index portfolio and retrieval representations

- **HGraph** is the flagship dense graph index for high-recall, low-latency search. It grew from
  the initial [HGraph implementation](https://github.com/antgroup/vsag/pull/114) into an index
  with quantization, filtering, range and iterator search, updates, mark/force removal, cache
  import/export, diagnostics, and memory-plus-disk configurations.
- **IVF** provides partition-based retrieval for large corpora, batch queries, and large `topk`
  workloads. It supports quantization, reordering, attribute filters, parallel build/search, and
  on-disk bucket storage. See the original [IVF PR](https://github.com/antgroup/vsag/pull/505).
- **Pyramid** supports path-scoped and multi-tenant retrieval. VSAG 1.0 adds named
  [multi-hierarchy build, add, and search](https://github.com/antgroup/vsag/pull/2226).
- **BruteForce** remains the exact-search baseline and an exact option for smaller collections.
  It also hosts the final multi-vector mode after the earlier WARP work was
  [consolidated into BruteForce](https://github.com/antgroup/vsag/pull/2275).
- **LazyHGraph** starts with exact BruteForce search and converts to HGraph after a configurable
  threshold, avoiding graph-build overhead for small, growing collections
  ([PR #2151](https://github.com/antgroup/vsag/pull/2151)).
- **SINDI** is the sparse-vector index for BM25-style and learned sparse retrieval. The v1.0 line
  adds term-ID remapping, analysis support, immutable reads, FP16 sparse values, compaction, and a
  [low-memory immutable build](https://github.com/antgroup/vsag/pull/2424).
- **Multi-vector retrieval** is represented directly in `Dataset` through the
  [MultiVector API](https://github.com/antgroup/vsag/pull/1995). In addition to BruteForce
  multi-vector mode, v1.0 introduces the specialized
  [SIMQ index](https://github.com/antgroup/vsag/pull/2357).

SINDI can serve as the sparse leg of a dense+sparse retrieval pipeline. VSAG 1.0 does not yet
provide built-in fused dense+sparse scoring; applications combine the result sets outside VSAG.

See [Indexes](../../indexes/) for the actively documented index families.

### Quantization, data types, and hardware acceleration

The `0.x` series built a broad memory/recall toolbox, and v1.0 brings those paths together:

- FP32, INT8, FP16, and BF16 dense inputs, plus sparse and multi-vector datasets.
- SQ4/SQ8 and their uniform variants for scalar quantization.
- Product Quantization and PQ FastScan, introduced in
  [PR #626](https://github.com/antgroup/vsag/pull/626) and
  [PR #691](https://github.com/antgroup/vsag/pull/691).
- RaBitQ, extended-bit and x+y split base/reorder layouts, FHT/PCA-assisted transforms, and
  dedicated SIMD kernels.
- Transform Quantizer chains and MRL-E dimension reduction.
- x86_64 dispatch across SSE, AVX, AVX2, AVX-512, and selected AMX kernels, plus ARM NEON and SVE.

VSAG 1.0 also accepts FP16/BF16 input vectors directly
([PR #1731](https://github.com/antgroup/vsag/pull/1731)) and adds AMX paths for SQ8U inner product
and KMeans BF16 GEMM ([PR #2032](https://github.com/antgroup/vsag/pull/2032)).

See [Quantization](../../quantization/) for supported combinations and tuning guidance.

### Search API, filtering, lifecycle, and observability

- `SearchRequest` and `Index::SearchWithRequest` provide one request object for KNN/range mode,
  JSON parameters, `FilterPtr`/bitset/attribute filtering, and reasoning inputs. Optional fields
  and modes are index-specific. The v1.0 implementation also
  [unifies common search paths](https://github.com/antgroup/vsag/pull/2341) across HGraph, IVF,
  and BruteForce.
- SQL-like attribute expressions and inverted attribute indexes support structured filtering in
  HGraph, IVF, and BruteForce. HGraph also supports iterator filtering. When
  `hgraph.brute_force_threshold` is configured, it can use a brute-force fallback for extremely
  selective filters.
- Index lifecycle operations now include `Train`, `Clone`, `ExportModel`, `Tune`, batch removal,
  mark/force remove modes, updates, source IDs, detail inspection, and memory estimation where the
  selected index advertises support.
- Reasoning data was added to `SearchRequest`, `Dataset`, and `QueryContext`
  ([PR #1838](https://github.com/antgroup/vsag/pull/1838)), then integrated into HGraph, IVF, and
  BruteForce. Search, I/O, memory, and index-specific statistics make offline diagnosis easier.
- `extra_info` payloads, index introspection, `CalcDistanceById` (with the older
  `CalDistanceById` name retained for compatibility), and analysis tools expose more of the index
  state without requiring an application-specific fork.

The original KNN, range, and filter overloads remain available in v1.0, so applications can adopt
`SearchRequest` incrementally.

### Storage, I/O, serialization, and compatibility

The v0.x releases introduced the DataCell and I/O abstractions, buffer/direct/async/mmap/reader
backends, on-disk IVF, and memory-plus-disk HGraph configurations. VSAG 1.0 adds a header-first,
forward-only [streaming serialization format](https://github.com/antgroup/vsag/pull/2256):

- `SerializeStreaming` writes metadata followed by typed TLV blocks.
- `DeserializeStreaming` restores into an existing empty, compatible index object.
- `Index::Load` reads metadata, creates the matching index, and applies supported placement policy.
- BruteForce, HGraph, IVF, mutable SINDI, and Pyramid implement the v1.0 streaming path. Immutable
  SINDI runtime serialization is not supported by this path in v1.0.

The new streaming format is **not compatible** with the existing `Serialize`/`Deserialize` format.
Both API families remain available, but a file must be read by the matching family. See
[New Serialization](../../advanced/new_serialization.md) for format and block-version details.

Cross-version index fixtures and the
[compatibility check tool](../check_compatibility.md) provide a repeatable way to validate old
artifacts before an upgrade.

### Platforms, bindings, build, and tools

- macOS gained [build support](https://github.com/antgroup/vsag/pull/1439), LLVM 15
  [format/lint compatibility](https://github.com/antgroup/vsag/pull/1906), and
  [PR CI coverage](https://github.com/antgroup/vsag/pull/1930). Linux x86_64 and AArch64 are
  covered by CI and Python wheel workflows; available artifacts vary by binding and architecture.
- The Python build moved to native CMake integration and added Python 3.13/3.14 support
  ([PR #1599](https://github.com/antgroup/vsag/pull/1599)). `pyvsag` also gained broader index
  operations, FP16/BF16 inputs, sparse-vector support, and sparse HDF5 helpers.
- VSAG now includes C APIs and
  [Node.js/TypeScript bindings](https://github.com/antgroup/vsag/pull/1812) with quickstart
  examples. Binding package versions and distribution channels should be checked independently
  from the core C++ tag.
- Builds can use system OpenBLAS/fmt dependencies, custom third-party download mirrors, installed
  CMake package metadata, and bundled release artifacts.
- `eval_performance` supports dense, sparse, and multi-vector datasets. `analyze_index`,
  `check_compatibility`, streaming visualization, and the HTTP monitor improve operational
  workflows.

### Reliability and engineering quality

The official v1.0.0 log contains 105 bug fixes covering concurrent search/update/remove, exception
safety and leaks, integer overflow and bounds checks, I/O cleanup, serialization state,
quantization edge cases, deterministic filtering, and platform-specific builds.

The release cycle also expanded ASan/TSan coverage, compatibility fixtures, unit coverage,
reproducible test data, public-header self-containment, layered source organization, and targeted
CI paths for documentation and changed files.

## Compatibility and Upgrade Notes from v0.18

VSAG 1.0 is a major release and includes source-level API changes. Review these points before
upgrading:

1. **`Remove` returns a count and supports batches.** The v0.18
   `tl::expected<bool, Error> Remove(int64_t)` API became
   `tl::expected<uint32_t, Error>` and gained a vector overload and explicit remove modes
   ([PR #1551](https://github.com/antgroup/vsag/pull/1551)). The final v1.0 API exposes
   `RemoveMode::MARK_REMOVE` and `RemoveMode::FORCE_REMOVE`; HGraph force removal landed in
   [PR #1810](https://github.com/antgroup/vsag/pull/1810).
2. **Unsupported operations usually return an error.** Many default methods that return
   `tl::expected` now return `tl::unexpected` with
   `ErrorType::UNSUPPORTED_INDEX_OPERATION` instead of throwing `std::runtime_error`
   ([PR #2141](https://github.com/antgroup/vsag/pull/2141)). Check the expected value before
   calling `.value()`.
3. **Memory-statistics signatures changed.** `GetMemoryUsage` uses `uint64_t`,
   `GetMemoryUsageDetail` returns a `std::unordered_map<std::string, uint64_t>`, and
   `GetEstimateBuildMemory` became `EstimateBuildMemory`
   ([PR #2388](https://github.com/antgroup/vsag/pull/2388)).
4. **Search migration can be incremental.** Prefer `SearchRequest` /
   `SearchWithRequest` for new integration work, but the existing search overloads remain in
   v1.0.
5. **Do not mix serialization families.** Legacy output must use legacy deserialization; streaming
   output must use `DeserializeStreaming` or `Index::Load`.
6. **SINDI heap insertion is automatic.** The legacy `use_term_lists_heap_insert` search parameter
   is ignored. SINDI derives the strategy from `doc_prune_ratio` and `query_prune_ratio`; update
   configurations that forced the old path.
7. **Intel MKL is opt-in.** The default is `OFF`. Enable it with
   `VSAG_ENABLE_INTEL_MKL=ON` through the Makefile or `-DENABLE_INTEL_MKL=ON` through CMake when
   required.

For persisted indexes, validate the exact source/target pair in staging. Serialization
compatibility can differ by index, feature flags, and format family.

## The v0.x Journey to 1.0

The current repository has no published tags before `v0.11.0`, so this history starts at the first
verifiable public release.

### Foundations: v0.11-v0.14

- **[v0.11](https://github.com/antgroup/vsag/releases/tag/v0.11.0), September 2024:** established
  the initial HNSW/DiskANN, C++/Python, pre-filter, cosine, locking, and serialization baseline.
- **[v0.12](https://github.com/antgroup/vsag/releases/tag/v0.12.0), December 2024:** introduced
  DataCell, I/O, and graph abstractions; HGraph; SQ4/SQ8/INT8 paths; the Engine/factory model; and
  pyvsag packaging.
- **[v0.13](https://github.com/antgroup/vsag/releases/tag/v0.13.0), February 2025:** added
  BruteForce, expanded Pyramid, memory estimation, index feature discovery, filter hints, and the
  `eval_performance` tool.
- **[v0.14](https://github.com/antgroup/vsag/releases/tag/v0.14.0), April 2025:** introduced IVF,
  FP16/BF16 and RaBitQ support, async/buffer I/O, sparse datasets, HGraph `extra_info`, iterator
  filtering, and systematic compatibility checks.

### Production breadth: v0.15-v0.18

- **[v0.15](https://github.com/antgroup/vsag/releases/tag/v0.15.0), June 2025:** added
  Train/Clone/ExportModel, PQ/PQ FastScan, attribute expressions, compressed graphs, HGraph merge
  and mark-delete, and self-describing serialization with compatibility CI.
- **[v0.16](https://github.com/antgroup/vsag/releases/tag/v0.16.0), August 2025:** added mmap
  HGraph, SINDI, parallel IVF, attribute updates, raw-vector access, parameter compatibility
  checks, and a long-running patch line for ABI, concurrency, and old-index fixes.
- **[v0.17](https://github.com/antgroup/vsag/releases/tag/v0.17.0), October 2025:** expanded
  `SearchRequest` to cover the main search cases and added search timeouts, broader `extra_info`,
  transformer quantization, single-query parallel HGraph search, export APIs, and richer SINDI
  lifecycle/statistics.
- **[v0.18](https://github.com/antgroup/vsag/releases/tag/v0.18.0), January 2026:** added the C API,
  automated Python wheels, sparse pybind support, on-disk IVF, index detail/search/I/O statistics,
  MRL-E and HGraph tuning, extended RaBitQ, and further SINDI/Pyramid maturity.

See the [v0.11.0...v1.0.0 comparison](https://github.com/antgroup/vsag/compare/v0.11.0...v1.0.0)
for the complete commit history.
