# VSAG 1.0 版本日志

`v1.0.0` 发布于 2026 年 7 月 12 日。

- [GitHub 官方 Release](https://github.com/antgroup/vsag/releases/tag/v1.0.0)
- [v0.18.0...v1.0.0 完整变更](https://github.com/antgroup/vsag/compare/v0.18.0...v1.0.0)
- Tag commit：`efdaf17a10e96cdb5222baf558d50dfacbdc672e`

## 版本概览

VSAG 1.0 是项目首个稳定大版本。
公开的 `0.x` 历史横跨 `v0.11`-`v0.18` 维护线，共发布 81 个 tags。
`v1.0.0` tag 记录了主线最终状态，形成覆盖稠密向量、稀疏向量、
结构化过滤、层级检索和多向量检索的广泛工具箱。

`v1.0.0` 官方 Release 将 `v0.18.0` 以来的工作归纳为 375 项：
48 项新增功能、134 项改进、105 项缺陷修复和 88 项其他变化。
`v0.11.0...v1.0.0` 主线对比包含 1,252 个 commits。

本文以 `v1.0.0` tag 的最终状态为准。
开发过程中曾加入、但在正式发布前又移除的中间功能，
不会作为 1.0 能力列出。

## VSAG 1.0 提供了什么

### 索引体系与检索表示

- **HGraph** 是面向高召回、低延迟稠密检索的旗舰图索引。它从最初的
  [HGraph 实现](https://github.com/antgroup/vsag/pull/114)逐步扩展出量化、过滤、
  范围检索、迭代检索、更新、标记/强制删除、
  缓存导入导出、诊断，以及内存+磁盘配置。
- **IVF** 面向大规模数据、批量查询和大 `topk` 场景，支持量化、reorder、属性
  过滤、并行构建/检索和磁盘 bucket 存储。参见最初的
  [IVF PR](https://github.com/antgroup/vsag/pull/505)。
- **Pyramid** 面向按路径限定范围的多租户与层级检索。VSAG 1.0 新增命名
  [多层级构建、写入与检索](https://github.com/antgroup/vsag/pull/2226)。
- **BruteForce** 继续作为精确检索基线，也是小规模集合的精确检索选项。
  早期 WARP 的多向量工作在正式发布前已
  [合并进 BruteForce](https://github.com/antgroup/vsag/pull/2275)。
- **LazyHGraph** 在数据量较小时使用精确 BruteForce，达到可配置阈值后转换为
  HGraph，适合持续增长但初始规模较小的集合
  （[PR #2151](https://github.com/antgroup/vsag/pull/2151)）。
- **SINDI** 面向 BM25 风格和学习型稀疏表示。1.0 系列新增 term ID 重映射、
  analyze、immutable read、稀疏 FP16 值、紧凑存储，以及
  [低内存 immutable build](https://github.com/antgroup/vsag/pull/2424)。
- **多向量检索**通过
  [MultiVector API](https://github.com/antgroup/vsag/pull/1995)成为 `Dataset`
  的直接表示。除 BruteForce 多向量模式外，v1.0 还引入专用
  [SIMQ 索引](https://github.com/antgroup/vsag/pull/2357)。

SINDI 可以作为稠密+稀疏检索管线中的稀疏一路。
VSAG 1.0 尚未内置稠密与稀疏分数融合，
应用需要在 VSAG 外部合并两路结果。

当前重点维护的索引族参见[索引](../../indexes/)。

### 量化、数据类型与硬件加速

`0.x` 系列逐步建立了广泛的内存/召回工具箱，
v1.0 将这些能力汇总到统一体系中：

- FP32、INT8、FP16、BF16 稠密输入，以及稀疏和多向量数据集；
- SQ4/SQ8 及对应 uniform 变体；
- Product Quantization 与 PQ FastScan，分别由
  [PR #626](https://github.com/antgroup/vsag/pull/626)和
  [PR #691](https://github.com/antgroup/vsag/pull/691)引入；
- RaBitQ、扩展 bit 与 x+y split base/reorder 布局、
  FHT/PCA 变换及专用 SIMD kernel；
- Transform Quantizer 链和 MRL-E 降维；
- x86_64 上的 SSE、AVX、AVX2、AVX-512 与部分 AMX kernel，以及 ARM NEON、SVE。

VSAG 1.0 还支持直接输入 FP16/BF16 向量
（[PR #1731](https://github.com/antgroup/vsag/pull/1731)），并增加 SQ8U 内积与
KMeans BF16 GEMM 的 AMX 路径
（[PR #2032](https://github.com/antgroup/vsag/pull/2032)）。

支持组合与调参建议参见[量化](../../quantization/)。

### 检索 API、过滤、生命周期与可观测性

- `SearchRequest` 与 `Index::SearchWithRequest` 使用一个请求对象表达
  KNN/range 模式、JSON 参数、`FilterPtr`/bitset/属性过滤和 reasoning 输入。
  可选字段与模式的支持程度取决于具体索引。
  v1.0 还在 HGraph、IVF、BruteForce 之间
  [统一了通用检索路径](https://github.com/antgroup/vsag/pull/2341)。
- SQL 风格属性表达式与属性倒排索引支持 HGraph、IVF、BruteForce
  的结构化过滤。HGraph 还支持迭代过滤。
  配置 `hgraph.brute_force_threshold` 后，
  它可以在过滤选择率极低时回退到暴力检索。
- 索引生命周期能力扩展到 `Train`、`Clone`、`ExportModel`、`Tune`、批量删除、
  标记/强制删除、更新、Source ID、细节读取与内存估算；
  具体能力以索引 feature 声明为准。
- Reasoning 信息被加入 `SearchRequest`、`Dataset` 与 `QueryContext`
  （[PR #1838](https://github.com/antgroup/vsag/pull/1838)），并逐步接入 HGraph、
  IVF、BruteForce。检索、I/O、内存和索引专用统计让离线诊断更加直接。
- `extra_info`、索引内省、`CalcDistanceById`
  （旧 `CalDistanceById` 名称仍为兼容性保留）与分析工具，
  让应用无需维护私有 fork 也能读取更多索引状态。

原有 KNN、range 和 filter 重载在 v1.0 中仍然保留，应用可以渐进迁移到
`SearchRequest`。

### 存储、I/O、序列化与兼容性

v0.x 版本逐步引入 DataCell 与 I/O 抽象、buffer/direct/async/mmap/reader 后端、
on-disk IVF，以及 HGraph 的内存+磁盘配置。VSAG 1.0 新增 header-first、只需前向
读取的[流式序列化格式](https://github.com/antgroup/vsag/pull/2256)：

- `SerializeStreaming` 先写 metadata，再写 typed TLV blocks；
- `DeserializeStreaming` 将数据恢复到已创建、为空且兼容的索引对象；
- `Index::Load` 读取 metadata、创建对应索引并应用支持的存储放置策略；
- BruteForce、HGraph、IVF、mutable SINDI、Pyramid
  已实现 v1.0 streaming 路径。
  Immutable SINDI runtime 在 v1.0 中不支持该序列化路径。

新 streaming 格式与既有 `Serialize`/`Deserialize` 格式**不兼容**。两套 API 都会
继续保留，但文件必须由配套的 API 家族读取。格式和 block version 细节参见
[新序列化格式](../../advanced/new_serialization.md)。

跨版本索引 fixtures 与
[兼容性检查工具](../check_compatibility.md)
提供了可重复执行的旧索引升级验证方式。

### 平台、绑定、构建与工具

- macOS 新增[构建支持](https://github.com/antgroup/vsag/pull/1439)、
  LLVM 15 [格式化/静态检查兼容](https://github.com/antgroup/vsag/pull/1906)，
  以及 [PR CI 覆盖](https://github.com/antgroup/vsag/pull/1930)。
  Linux x86_64 与 AArch64 均有 CI 和 Python wheel workflow 覆盖；
  可用产物因绑定和架构而异。
- Python 构建迁移到原生 CMake 集成，并支持 Python 3.13/3.14
  （[PR #1599](https://github.com/antgroup/vsag/pull/1599)）。`pyvsag` 还扩展了
  索引操作、FP16/BF16 输入、稀疏向量与稀疏 HDF5 helper。
- VSAG 新增 C API 与
  [Node.js/TypeScript 绑定](https://github.com/antgroup/vsag/pull/1812)，并提供
  quickstart 示例。绑定包版本与分发渠道需要独立于 C++ core tag 进行确认。
- 构建支持系统 OpenBLAS/fmt、自定义第三方下载镜像、安装后的 CMake package
  metadata，以及打包后的 release assets。
- `eval_performance` 支持稠密、稀疏和多向量数据集；`analyze_index`、
  `check_compatibility`、streaming 可视化与 HTTP monitor 完善了运维工具链。

### 稳定性与工程质量

v1.0.0 官方日志包含 105 项缺陷修复，
覆盖并发检索/更新/删除、异常安全与内存泄漏、
整数溢出与越界、I/O 资源清理、序列化状态、
量化边界、确定性过滤和平台构建。

发布周期还扩展了 ASan/TSan、兼容性 fixtures、
单测覆盖率、可复现测试数据、公共头文件自包含、
单向源码分层，以及针对文档和变更文件的精细 CI 路径。

## 从 v0.18 升级的兼容性说明

VSAG 1.0 是大版本升级，包含源码级 API 变化。升级前请重点检查：

1. **`Remove` 返回删除数量并支持批量。** v0.18 的
   `tl::expected<bool, Error> Remove(int64_t)` 调整为
   `tl::expected<uint32_t, Error>`，并增加 vector 重载和显式删除模式
   （[PR #1551](https://github.com/antgroup/vsag/pull/1551)）。
   v1.0 最终 API 提供 `RemoveMode::MARK_REMOVE` 和
   `RemoveMode::FORCE_REMOVE`；HGraph force remove 由
   [PR #1810](https://github.com/antgroup/vsag/pull/1810)实现。
2. **不支持的操作通常改为返回错误。**
   许多返回 `tl::expected` 的默认方法不再抛出
   `std::runtime_error`，而是返回带
   `ErrorType::UNSUPPORTED_INDEX_OPERATION` 的 `tl::unexpected`
   （[PR #2141](https://github.com/antgroup/vsag/pull/2141)）。调用 `.value()` 前
   应先检查 expected。
3. **内存统计接口签名发生变化。** `GetMemoryUsage` 使用 `uint64_t`，
   `GetMemoryUsageDetail` 返回 `std::unordered_map<std::string, uint64_t>`，
   `GetEstimateBuildMemory` 更名为 `EstimateBuildMemory`
   （[PR #2388](https://github.com/antgroup/vsag/pull/2388)）。
4. **检索接口可以渐进迁移。** 新接入优先使用 `SearchRequest` /
   `SearchWithRequest`，既有检索重载在 v1.0 中仍然保留。
5. **不要混用两套序列化格式。**
   旧格式输出必须使用旧反序列化接口；
   streaming 输出必须使用 `DeserializeStreaming` 或 `Index::Load`。
6. **SINDI 自动选择 heap insertion。** 旧的 `use_term_lists_heap_insert` 检索参数
   会被忽略。SINDI 根据 `doc_prune_ratio` 和 `query_prune_ratio` 推导策略；依赖
   强制指定旧路径的配置需要调整。
7. **Intel MKL 改为显式开启。** 默认值为 `OFF`。通过 Makefile 构建时设置
   `VSAG_ENABLE_INTEL_MKL=ON`，直接使用 CMake 时设置
   `-DENABLE_INTEL_MKL=ON`。

对于持久化索引，
建议在测试环境验证明确的源版本与目标版本组合。
序列化兼容性可能因索引类型、feature flags 和格式家族而不同。

## 从 v0.x 走向 1.0

当前仓库在 `v0.11.0` 之前没有公开发布 tag，
因此本节从首个可验证的正式版本开始。

### 基础建设：v0.11-v0.14

- **[v0.11](https://github.com/antgroup/vsag/releases/tag/v0.11.0)，2024 年 9 月：**
  建立 HNSW/DiskANN、C++/Python、预过滤、cosine、锁与序列化的初始基线。
- **[v0.12](https://github.com/antgroup/vsag/releases/tag/v0.12.0)，2024 年 12 月：**
  引入 DataCell、I/O、Graph 抽象、HGraph、SQ4/SQ8/INT8、Engine/factory 与
  pyvsag 打包。
- **[v0.13](https://github.com/antgroup/vsag/releases/tag/v0.13.0)，2025 年 2 月：**
  新增 BruteForce，扩展 Pyramid、内存估算、IndexFeature、filter hint 与
  `eval_performance`。
- **[v0.14](https://github.com/antgroup/vsag/releases/tag/v0.14.0)，2025 年 4 月：**
  引入 IVF、FP16/BF16、RaBitQ、async/buffer I/O、稀疏数据、HGraph
  `extra_info`、迭代过滤与系统化兼容性检查。

### 生产能力扩展：v0.15-v0.18

- **[v0.15](https://github.com/antgroup/vsag/releases/tag/v0.15.0)，2025 年 6 月：**
  新增 Train/Clone/ExportModel、PQ/PQ FastScan、属性表达式、压缩图、HGraph
  merge/mark-delete，以及带兼容性 CI 的自描述序列化。
- **[v0.16](https://github.com/antgroup/vsag/releases/tag/v0.16.0)，2025 年 8 月：**
  新增 mmap HGraph、SINDI、并行 IVF、属性更新、raw vector、参数兼容性检查，并
  通过长期补丁线持续解决 ABI、并发和老索引兼容问题。
- **[v0.17](https://github.com/antgroup/vsag/releases/tag/v0.17.0)，2025 年 10 月：**
  扩展 `SearchRequest` 以覆盖主要检索场景，并增加检索超时、
  更广泛的 `extra_info`、transform quantizer、HGraph 单查询并行、
  数据导出与更完整的 SINDI 生命周期/统计。
- **[v0.18](https://github.com/antgroup/vsag/releases/tag/v0.18.0)，2026 年 1 月：**
  新增 C API、自动 Python wheel、稀疏 pybind、on-disk IVF、索引细节/检索/I/O
  统计、MRL-E 与 HGraph Tune、扩展 RaBitQ，并继续完善 SINDI/Pyramid。

完整提交历史参见
[v0.11.0...v1.0.0 对比](https://github.com/antgroup/vsag/compare/v0.11.0...v1.0.0)。
